tcx-investor-context

Interview and preview optional workspace-local suitability context so a future run can account for the constraints that matter to its user without exposing private details as a generic profile.

Discuss suitability context #

Use this skill to discuss objective, horizon, loss capacity, liquidity needs, concentration, tax, and account constraints. It returns proposed values, outstanding gaps, and the workspace’s default application state for confirmation.

Example

$tcx-investor-context
Help me define the optional suitability context for this workspace.

Keep persistent updates operator-owned #

The Codex conversation interviews and previews only. For persistent status, update, enable, disable, or clear work, the skill gives one exact user-terminal command after confirmation. It does not smuggle a persistent change through the model shell.

Understand run binding #

The saved workspace default determines whether Investor Context is sealed into a run. The read-only viewer offers no one-run override and does not rewrite the file. Its private body is not a source of broker authority, and only its sealed provenance can be recorded with a run.

Suitability is not authority. Investor Context does not run investment analysis, store secrets, grant execution authority, or change broker and policy gates.
