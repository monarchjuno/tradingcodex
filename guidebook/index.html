Introduction

Invest with a clear question, not a complicated setup.

TradingCodex is a local workspace for evidence-led investment research in Codex. Ask a question in plain language, keep the answer current and traceable, and add structure only when your process needs it.

Choose your starting path #

You do not need to learn every skill or build a system before getting value. Start with the left path; move to the right when you want work that is repeatable, reusable, or reviewed over time.

01

I want an answer quickly
Ask one focused question. TradingCodex gathers only the evidence the answer needs and keeps research separate from any order action.

02

I want a repeatable process
Turn your method into reusable context, review prior decisions, automate monitoring, or follow the protected provider-to-order path.

Set up once #

Create an empty folder for your research, then attach TradingCodex. You need Git, uvx, and an installed, authenticated Codex CLI. End users do not need Node or npm.

macOS or Linuxshell
cd /path/to/your/empty-workspace
uvx --refresh --from tradingcodex tcx attach . && ./tcx doctor
./tcx service ensure
Windows PowerShellpowershell
cd C:\path\to\your\empty-workspace
uvx --refresh --from tradingcodex tcx attach .
.\tcx.cmd doctor
.\tcx.cmd service ensure

Then fully restart Codex, open and trust the workspace, review the generated TradingCodex hooks when prompted, and start a new task. The optional viewer is read-only; its default Episodes page follows a run from synthesis through Judgment, optional user adoption, Forecast, Postmortem, and Lesson. Use Codex itself for research and managed skills.

Ask your first question #

Use $tcx-workflow for almost every first research task. State what you want to decide, your time horizon, the questions that matter, and limits that must remain in force. “No order” is a useful default.

Your first prompt

$tcx-workflow
Analyze MSFT as a medium-term quality compounder.
Include contrary evidence and invalidation conditions. No order.
  1. Ask naturally. You do not need to choose analysts, data sources, or a research template.
  2. Review the answer. It should be complete without opening a report and make the conclusion, evidence, as-of context, uncertainty, contrary evidence, readiness, and gaps visible. A saved report link preserves deeper provenance.
  3. Continue or stop. Ask a follow-up in the same task when it is about the same question. Start a new task for a fresh mandate or a separate managed capability.
Not sure what to ask? Start with $tcx-plan to clarify the outcome, scope, horizon, allowed actions, and stop conditions. Then bring that mandate to a new $tcx-workflow task.

What TradingCodex does for you #

01

Finds the smallest useful research path

Simple questions stay small. Broader questions receive only the specialist perspectives and source checks that improve the answer.

02

Keeps the decision legible

Conclusion-relevant evidence is current and attributable. The result distinguishes support, contrary evidence, uncertainty, and missing information.

03

Separates research from action

Research never places a trade. Provider setup, review, approval, and any final broker action remain deliberate, separate checkpoints.
