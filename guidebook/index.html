Introduction

Get started

Set up one local workspace, then start from the outcome you want. The detailed pages explain both the skills you invoke and the system that keeps work evidence-based and bounded.

Before you begin #

You need an empty folder for research, Git, uvx, and an installed, authenticated Codex CLI 0.144.1 or later. Version 0.144.4 is the current tested reference; check with codex --version. End users do not need Node or npm.

01

Choose a workspace
Keep research in a folder you own and can back up.

02

Attach TradingCodex
Generate the workspace contract and local launcher.

03

Open a new task
Use a skill as the first line when work needs a defined path.

Install a workspace #

Run the installer inside the empty folder where research should live. It creates local workspace files; it does not create a commit or remote.

macOS or Linuxshell
cd /path/to/your/empty-workspace
uvx --refresh --from tradingcodex tcx attach . && ./tcx doctor
./tcx service ensure
Windows PowerShellpowershell
cd C:\path\to\your\empty-workspace
uvx --refresh --from tradingcodex tcx attach .
.\tcx.cmd doctor
.\tcx.cmd service ensure

Open the read-only workspace viewer at http://127.0.0.1:48267/. To work in Codex, restart Codex, open and trust that workspace, review and trust all eight generated TradingCodex hooks when prompted, then start a new task.

Quick start #

Choose the outcome, not a role. Start every example in a new native Codex task from the trusted workspace.

01

Research a current question

Use tcx-workflow to turn an investment question into a dynamic evidence workflow. Use tcx-plan first only when the mandate needs to be made concrete.

Read the research skill guide →
02

Connect a provider, then use a protected order path

Start with the Paper provider or prepare a reviewed broker connection. Research, ticket, checks, approval, and final action remain separate checkpoints.

Follow the provider-to-order guide →
03

Learn from a past decision

Use tcx-memory to retrieve and replay past evidence. Create a reusable Strategy or local Investment Brain in its own capability-scoped task.

Read the improve skill guide →
04

Change the workspace deliberately

Use tcx-build for managed workspace work. Use tcx-investor-context to preview optional suitability context.

Read the customize skill guide →
Need the dashboard or status first? Use $tcx-dashboard to open the read-only viewer in Codex by default, or explicitly request an external browser. Use $tcx-server for readiness, update, MCP, and recovery guidance.
