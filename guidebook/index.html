Introduction

Get started

Set up one local workspace, ask a tightly scoped investment question, and keep the research, evidence, and next steps available after the task ends.

i
Start with research. TradingCodex is designed to help you inspect evidence and constraints. A natural-language answer never becomes a broker action on its own.

Before you begin #

You need an empty folder for research, Git, uvx, and an installed, authenticated Codex CLI. End users do not need Node or npm.

01

Choose a workspace
Keep research in a folder you own and can back up.

02

Attach TradingCodex
Generate the workspace contract and local launcher.

03

Open Work
Start with a question, not a preselected team.

Install a workspace #

Run the installer inside the empty folder where you want your research files to live. It initializes local workspace files; it does not create a commit or remote.

macOS or Linuxshell
cd /path/to/your/empty-workspace
uvx --refresh --from tradingcodex tcx attach . && ./tcx doctor
./tcx service ensure
Windows PowerShellpowershell
cd C:\path\to\your\empty-workspace
uvx --refresh --from tradingcodex tcx attach .
.\tcx.cmd doctor
.\tcx.cmd service ensure

Open the local workbench at http://127.0.0.1:48267/. To use the workspace in the Codex app, restart Codex, open and trust that workspace, then start a new task.

Quick start with skills #

In a new Codex task opened from your trusted workspace, place one skill invocation on the first line. Then write the request in ordinary language. Start with these four paths.

01

Research a current question

Use tcx-workflow for company research, a thesis review, portfolio fit, risk, or a Decision Package. It chooses the smallest useful specialist team for the question.

$tcx-workflow
Analyze NVDA. Focus on business quality and contrary evidence. No order.
02

Retrieve or replay a past decision

Use tcx-memory when the question is about what was known before, a changed thesis, a forecast outcome, or a lesson.

$tcx-memory
Replay my latest NVDA decision using point-in-time evidence. What changed?
03

Clarify a high-stakes request

Use tcx-plan when scope, time horizon, blocked actions, or stop conditions are unclear. Carry the resulting mandate into a new tcx-workflow task.

$tcx-plan
Help me scope a review of a possible NVDA position. No order drafting.
04

Check workspace health

Use tcx-server for service readiness, update status, or safe recovery guidance. It does not run investment analysis.

$tcx-server
Is this workspace ready for a new analysis?
↳
Let the workflow select specialists. Do not start with role-owned skills such as tcx-fundamental or tcx-portfolio; tcx-workflow dispatches those only when the request needs them.

Run your first workflow #

Use tcx-workflow for an immediate research request. State the question, time horizon, evidence you need, and actions that are outside the task.

Starter prompttext
$tcx-workflow
Analyze NVDA. Focus on business quality, price action, recent disclosures,
and contrary evidence. No order, no trading, no valuation.
Include

The question, time horizon, evidence you want, and what could change your mind.

Exclude

Say “no order,” “no trading,” or “no valuation” whenever those are outside the task.

Prompt recipes #

Copy one of these into a new Codex task, then replace the ticker, horizon, or context.

Company research⌄
$tcx-workflow
Analyze MSFT as a medium-term quality compounder. Separate facts, inferences, and assumptions. Include contrary evidence and invalidation conditions. No order.
Challenge a thesis⌄
$tcx-workflow
Revisit my latest NVDA thesis. Focus on stale sources, the strongest contrary evidence, overconfidence risk, and what would change the conclusion. No trading.
Review portfolio fit⌄
$tcx-workflow
Review whether TSLA fits my current portfolio context. Cover concentration, liquidity, downside, and opportunity cost. Stop before order drafting.
Replay a past decision⌄
$tcx-memory
Replay my latest decision on NVDA with point-in-time evidence. Identify what was known, what changed, and the next validation needed.

Follow a run #

Head Manager starts a lightweight run and selects the smallest useful specialist team as accepted evidence arrives. Watch the status instead of treating the final report as a black box.

waiting

Required evidence or an artifact has not arrived yet.

revise

The owning specialist needs to improve an artifact.

blocked

Evidence, scope, or policy prevents safe progress.

accepted

The handoff cleared its quality gate and can support the next step.

Use the Library #

Once a handoff is accepted, open Library to read the report with its sources, snapshots, versions, and forecasts. Use that material to ask a narrower follow-up or begin a new attempt.

↳
Keep work portable. Research remains in ordinary workspace files under trading/research/, trading/reports/, and trading/forecasts/, so it can be read, backed up, diffed, and versioned with normal tools.

Choose a skill when work repeats #

Plain language is the normal entry point. Invoke a skill when you need a bounded, repeatable workflow.

GoalStart with
Research a company, thesis, portfolio fit, or risk questiontcx-workflow
Replay a decision or validate a lessontcx-memory
Design a reusable investing methodtcx-strategy
Set optional workspace suitability contexttcx-investor-context
Check service and workspace healthtcx-server

See the user-facing skills reference for every entry rule and hard stop.

Keep the request bounded #

Use explicit exclusions when you only want research. The workbench is analysis-only: it cannot draft, approve, cancel, or execute an order. If a run stops at waiting, revise, or blocked, inspect the reason before asking a broader question.

Troubleshooting #

Run these inside an attached workspace. On native Windows, substitute .\tcx.cmd for ./tcx.

./tcx doctorCheck the full workspace and service contract.
./tcx service ensureStart or reuse the local workbench service.
./tcx skills list --allInspect built-in, optional, strategy, active, and archived skills.
./tcx workspace statusInspect workspace identity and runtime provenance.
