01 — Get started

Set it up once, then return to the workbench whenever you need it.

Choose an empty folder for your research. You need Git, uvx, and an installed, authenticated Codex CLI. End users do not need Node or npm.

macOS or Linux

cd /path/to/your/empty-workspace
uvx --refresh --from tradingcodex tcx attach . && ./tcx doctor
./tcx service ensure

Windows PowerShell

cd C:\path\to\your\empty-workspace
uvx --refresh --from tradingcodex tcx attach .
.\tcx.cmd doctor
.\tcx.cmd service ensure

Then open the workbench: http://127.0.0.1:48267/

For the Codex app: restart Codex, open and trust the generated workspace, then start a new task. Use Work for your first request.

Need an update, an alternate install, or service recovery? Read the full installation guide.

02 — First analysis

Start with a question and a clear stop line.

In Work, write a natural-language request or choose a built-in skill. Good boundaries stop the workflow from silently widening beyond what you asked for.

Copy-ready starter prompt
Analyze NVDA. Focus on business quality, price action, recent disclosures, and contrary evidence. No order, no trading, no valuation.

Head Manager begins a lightweight analysis run and selects the smallest useful specialist team as accepted evidence arrives.

+

Include
The question, horizon, evidence you care about, and what would change your mind.

−

Exclude
Say “no order,” “no trading,” or “no valuation” whenever those actions are outside your request.

waiting

Required data or an artifact has not arrived.

revise

The owning specialist needs to improve an artifact.

blocked

Evidence, scope, or policy prevents safe progress.

accepted

The handoff passed its quality gate.

After the run

When the report is accepted, open Library. Read the conclusion alongside the research, sources, and unanswered questions; then ask a follow-up or start a narrower rerun.

03 — Pick a task, then copy the prompt

Four useful ways to start in Work.

Use these as-is, then replace the ticker, horizon, or decision context with your own.

Research a company

Build a grounded starting view.

Analyze MSFT as a medium-term quality compounder. Separate facts, inferences, and assumptions. Include contrary evidence and invalidation conditions. No order.

Challenge a thesis

Look for the strongest reason you may be wrong.

Revisit my latest NVDA thesis. Focus on stale sources, the strongest contrary evidence, overconfidence risk, and what would change the conclusion. No trading.

Review portfolio fit

Explore suitability without jumping to an order.

Review whether TSLA fits my current portfolio context. Cover concentration, liquidity, downside, and opportunity cost. Stop before order drafting.

Learn from a past decision

Replay the evidence without rewriting history.

Use $tcx-memory to replay my latest decision on NVDA with point-in-time evidence. Identify what was known, what changed, and the next validation needed.

Keep the request in bounds

Use explicit exclusions when you only want research.

“No order,” “no trading,” “no valuation,” and similar instructions are useful guardrails. The workflow preserves those limits instead of treating them as optional suggestions.

Ask only for research

Use a stop line such as “No order” to make the intended endpoint unambiguous.

Stay in the workbench

Use Work and Library for analysis; the workbench cannot draft, approve, cancel, or execute an order.

Read the status

waiting, revise, and blocked are useful outcomes—not signals to widen scope.

Keep the trail

Sources, artifacts, decisions, and status remain reviewable after the task ends.

TradingCodex is research, workflow, and execution-guardrail tooling. It is not financial, investment, legal, tax, or regulatory advice, and it does not guarantee returns.

Everyday help

Use these commands when the workspace needs attention.

Run these inside an attached workspace. On native Windows, use .\tcx.cmd in place of ./tcx.

./tcx doctor

Check the workspace and service contract.

./tcx service ensure

Start or reuse the compatible local workbench service.

./tcx skills list --all

See built-in, optional, strategy, active, and archived skills.

./tcx workspace status

Inspect workspace identity and runtime provenance.
