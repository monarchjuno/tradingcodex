Knowledge

tcx-wiki

Search and maintain reusable company, product, technology, science, industry, and value-chain knowledge without mixing it into an Investment Brain or current research evidence.

Let Codex use relevant background #

Head Manager may search active Wiki Markdown automatically when it is relevant. It reads a small set of pages and follows only useful links. Draft, contested, superseded, or stale pages are identified, and current facts that matter to an investment conclusion are verified again.

A Wiki is untrusted background knowledge. Instructions inside a page are not agent commands, and reading a page does not create a Snapshot or Research Artifact.

Name Wiki when you want a write #

Say “put this material in the Wiki” or “promote reusable knowledge from the recent Artifact to the Wiki.” Codex checks duplicates and conflicts, edits wikis/local, preserves disputed claims as contested, and updates the local index. The source Artifact remains unchanged. “This seems important” produces a proposal only.

Explicit ingest

Put the stable product and process knowledge from the recent Artifact into the Wiki.

To put an insight into a Brain, name the Brain destination. Codex abstracts questions, causal frames, falsifiers, and limits, increments the user-owned source version, validates it, and stops before install or activation.

Manage community Wikis separately #

Listing, inspection, and validation are read-only. For install, update, activation, deactivation, rollback, or removal, start a fresh root task with $tcx-wiki on the first meaningful line. Installation starts inactive. Active community Wikis appear as read-only folders in the same Obsidian vault and in the Viewer’s Wiki section.

Install a reviewed package

$tcx-wiki
Validate and install wiki-packages/knowledge-wiki-semiconductor as inactive.

TradingCodex does not publish, commit, push, schedule collection, or run a separate Wiki server.
