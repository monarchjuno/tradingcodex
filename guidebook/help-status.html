Open and check the workspace without changing it

Use the dashboard skill to open the read-only viewer and orient to current workspace state, and the server skill for readiness, update, MCP, and recovery diagnostics.

Read the skill pages #

Primary

tcx-dashboard

Open the read-only workspace dashboard and orient to attention items, recent work, portfolio/order posture, forecasts, permissions, and broker state.

Use it when
You want to open the dashboard, know what needs attention, see where work stands, or inspect a workspace domain.
What it does
Opens the Codex in-app browser by default, uses an external browser only when explicitly requested, and reads only the smallest relevant canonical status and ledger surfaces.
What it returns
The opened Library, Skills, System, or base dashboard destination plus a compact recorded-state orientation.
Boundary
It does not begin analysis, dispatch roles, create artifacts, make recommendations, change TradingCodex state, or use the shell to launch a browser.

Open in Codex

$tcx-dashboard
Open the workspace dashboard.
Primary

tcx-server

Explain workspace and service health, update readiness, MCP state, database posture, and recovery guidance.

Use it when
The workspace is not ready, you need the viewer URL, MCP needs checking, or you want a safe recovery and update posture before a new task.
What it does
Uses the smallest trusted read-only status surface and interprets hook startup context plus read-only status/update tools.
What it returns
Runtime status, a recovery command, a viewer URL, update guidance, or a clearly stated blocker.
Boundary
It does not perform investment judgment, run service commands through the model shell, modify connector code, or make a workspace change.

Check readiness

$tcx-server
Is this workspace ready for a new analysis?

Use the terminal for explicit operations #

When a service, doctor, update, or recovery command requires the launcher, the skill gives you the exact user-terminal handoff rather than running it through the model shell. That keeps maintenance operations clearly operator-owned.

Common readiness checkshell
./tcx doctor
./tcx service ensure

The default doctor view shows layer totals and expands warnings or failures. Use ./tcx doctor --verbose when you need every PASS check. A package refresh launched with uvx ... tcx update preserves the workspace's recorded custom runtime home, DB override, and service address unless you explicitly provide replacements. On native Windows PowerShell, use .\tcx.cmd doctor, .\tcx.cmd doctor --verbose, and .\tcx.cmd service ensure.

Updating to 1.1.1. Pin both sides of the update with uvx --refresh --from "tradingcodex==1.1.1" tcx update . --from "tradingcodex==1.1.1", then run ./tcx doctor. On native Windows PowerShell, run the same pinned uvx command, then run .\tcx.cmd doctor. Fully quit and restart Codex afterward and open a new task. User-installed MCP servers, skills, and plugins remain Codex-owned BYOR capabilities; TradingCodex refreshes only its reserved projections.

What the viewer can show #

The local workspace viewer is a read-only projection. It can show Library, Skills, and System information for registered attached workspaces, but it never launches or resumes Codex, creates an analysis run, modifies a skill, or offers a browser exception to policy and execution boundaries.

Use a new native Codex task for agent work. The viewer is where you inspect the workspace; the trusted Codex workspace is where research and managed skills run.
