Primary skill

tcx-workflow

Start an evidence-led investigation from a natural-language investment task. This is the default entrypoint for current research, thesis review, portfolio fit, risk review, and Decision Packages.

Describe the outcome and constraints #

Start a new native Codex task and invoke $tcx-workflow, either as the plain token or its matching workspace skill link. Then write the outcome you need, the time horizon, the questions that matter, and the limits that must remain in force. Natural language is expected; exclusions such as “No valuation” and “No order” are binding.

Example

$tcx-workflow
Analyze MSFT as a medium-term quality compounder. Include contrary evidence and invalidation conditions. No order.

What happens during the run #

  1. Begin a lightweight run. The request and optional selected context receive sealed provenance.
  2. Orient when needed. Head Manager may use narrow live-web reconnaissance to resolve the subject, event, source landscape, and material unknowns. Raw results remain untrusted planning leads.
  3. Choose the smallest useful team. Head Manager selects exact fixed-role specialists from the current task, not a preset analyst roster. Conclusion-relevant facts must be reacquired by those roles.
  4. Retrieve progressively. Reusable data starts as a compact Dataset card, then manifest/profile metadata, then a bounded typed slice only when the owning specialist needs rows.
  5. Calculate reproducibly. A specialist stages one scratch-local script and uses the pinned tcx-calc runtime. Conclusion-relevant work is prepared and recorded against exact inputs, parameters, cutoff, output schema, and runtime; direct unprepared runs remain exploratory.
  6. Use authenticated artifacts. Each specialist returns a bounded report with exact Dataset and Calculation bindings; new evidence can trigger a challenge, a new perspective, or a stop.
  7. Synthesize or wait. The run ends in a traceable synthesis, revise/blocked state, or waiting for the required role.

Read Dynamic workflow for the fixed-role team and artifact-driven reassessment model.

What you receive #

The result is a current evidence-based answer or Decision Package with source/as-of posture, uncertainty, contrary evidence, missing evidence, and a recorded handoff state. The exact set of artifacts depends on the question; a simple factual request can remain small.

Workflow is not execution. This skill does not create a broker action, and fixed-role specialists cannot submit or cancel one. Use an exact order entrypoint only when you deliberately intend a final-action path.
