Primary skill

tcx-dashboard

Open the read-only dashboard for the current attached workspace and get a compact orientation to attention items, recent research, forecasts, portfolio and order posture, pending permissions, and broker state.

Open the dashboard where you want it #

An unqualified invocation opens the dashboard in the Codex in-app browser. Ask explicitly for an external browser only when you want the dashboard outside Codex. If the requested surface is unavailable, the skill returns the exact clickable URL instead of silently switching browsers.

The viewer is optimized for a half-width desktop window: the workspace rail becomes a compact selector, and Library or Skills opens one full-width list or reader at a time. Wider windows retain the side-by-side reader, while phone layouts keep full-width controls and navigation.

Open in Codex

$tcx-dashboard
Open the workspace dashboard.

Open externally

$tcx-dashboard
Open the workspace dashboard in an external browser.

Read recorded state, not guesses #

The dashboard routes to #/library, #/skills, or #/system when a narrower destination is requested. Its compact orientation puts explicit blocked, waiting, stale, failed, unhealthy, pending, uncertain, or incompatible states first. It calls something changed only when a trusted comparison, version, event, or timestamp establishes that change; missing data remains unknown.

Keep overview separate from action #

Dashboard is not analysis or execution. It does not begin an analysis run, dispatch roles, create artifacts, make investment recommendations, mutate workspace or service state, or perform an order action.

Use tcx-workflow for a new investment judgment. Use tcx-server when service readiness or recovery is the problem.
