Primary skill

tcx-server

Check whether an attached workspace is ready, find its read-only viewer, understand update and MCP posture, and receive a safe recovery handoff when operator-terminal work is required.

Ask for the status you need #

Use this skill before a new analysis when the workspace may not be ready, when the viewer URL is unknown, or when service, database, update, and MCP posture need to be checked.

Example

$tcx-server
Is this workspace ready for a new analysis?

Use the trusted read-only result #

The skill uses hook startup context and the smallest trusted read-only status/update surface. It returns runtime status, a viewer URL, update guidance, a recovery command, or a clearly stated blocker rather than guessing.

If a launcher command is required, it is provided as an explicit user-terminal step. The skill does not run service, doctor, update, or recovery commands through the model shell.

Stay in the operations lane #

Status is not investment judgment or workspace modification. This skill does not analyze an investment, modify connector code, or create a service action.

The local browser viewer it references is a read-only projection. Use a new trusted native Codex task for agent work; use the terminal for explicit operational commands.
