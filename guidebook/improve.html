Learn, generalize, and refine

These skills let a workspace retain useful judgment over time. They keep past evidence, reusable rules, and inquiry heuristics distinct so that none quietly becomes an authority grant.

Primary

tcx-memory

Retrieve prior decisions, replay point-in-time evidence, review outcomes, and validate lessons.

Use it when
You need to know what was previously known, why a thesis changed, or whether a lesson holds across comparable cases.
What it preserves
Supporting and contrary episodes, retired claims, evidence origin, forecast/review records, and the difference between process and outcome quality.
Current judgment
It records the independent current-evidence view before comparable memory is introduced, then records the post-memory delta.
Boundary
Memory evidence alone cannot draft, approve, or execute an order, and it does not automatically change a Strategy or Brain.

Review a decision

$tcx-memory
Replay my latest NVDA decision using point-in-time evidence. What changed?

Scoped task

tcx-brain

Author user-owned sources and manage installed Investment Brain versions through one entrypoint.

Use it when
You want to create, inspect, revise, validate, install, update, activate, deactivate, roll back, or remove an Investment Brain.
Required posture
Start a new native trading-research task whose exact first line is $tcx-brain. Do not wrap it in $tcx-build.
What it returns
A privacy-reviewed source action or a canonical lifecycle result with id, version, status, digests, projection posture, and next step.
Boundary
The current-turn grant covers only Brain source and lifecycle paths. Installation starts inactive, activation and deletion stay explicit, managed removal retains provenance, and no action implies Git publication.

Create a local source bundle

$tcx-brain
Create a local Investment Brain from these selected lessons and counterexamples.

A Brain guides inquiry; it never selects roles, calls tools, owns workflow, retrieves or modifies memory, changes policy, or grants execution authority.

Scoped task

tcx-strategy

Design a reusable user Strategy with validated lifecycle and projection metadata.

Use it when
You want durable, reusable decision rules for later workflows: eligibility, entry/exit, sizing, risk rules, and explicit falsifiers.
Required posture
Start a new native trading-research task whose exact first line is $tcx-strategy. Do not wrap it in $tcx-build.
What it returns
A validated strategy-* skill with required sections, status, projection metadata, and user approval posture.
Boundary
The current-turn grant covers only Strategy lifecycle work. Activation, archive, and deletion stay explicit; a Strategy never approves orders, grants broker authority, mutates policy, or executes a trade.
