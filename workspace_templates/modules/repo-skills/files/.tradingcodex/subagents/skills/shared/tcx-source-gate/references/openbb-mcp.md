# Direct OpenBB MCP

Read this only after choosing the direct OpenBB route exposed to the current
task. Do not install, configure, provision, or start OpenBB.

- Discover and use only the needed route/tool; do not expand broad categories.
- Name the requested provider when supported, and verify the returned provider.
- Before relying on a response, verify the symbol or identifier, venue,
  period/as-of, currency, timezone, raw/adjusted policy, and warnings.
- Treat authentication, entitlement, rate-limit, empty, or stale responses as
  missing coverage, not proof that the data itself is absent.
- Keep a partial success and query the next source only for the uncovered part.
- Record the response as a SourceSnapshot immediately; create a Dataset when
  reusable structured rows are consumed.

Do not assume a tool name or OpenBB version. This remains a direct user-enabled
MCP, not a TradingCodex wrapper, supervisor, provider registry, or receipt.
